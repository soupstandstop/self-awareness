# 2026-04-28 — AI 拿走的，是實作還是思考？

## Android Bug：三小時 vs. 翻文件幾分鐘

今天你花了三小時讓 Claude Code debug Android stagingDebug，他始終找不到 root cause。最後你去翻了 project skill，問題就解決了。

這件事讓你有個念頭：**人類的本性就是懶惰。** AI 現在可以擔任 implementation & execute 的角色，這讓你很放心地把事情交出去——但同時你也感覺到，AI 正在潛移默化地把「思考」與「設計」的工作也一併拿走。

你越來越習慣把所有的事，包含思考本身，都請 AI 代勞。

---

## 這讓你感到恐懼

如果連日常自我思考的訓練都被 AI 取代，人類是否還有存在在世界上的意義？

人類優於 AI 的地方，似乎越來越少。

這個問題你還沒有答案，但你知道它值得被認真對待，而不是迴避。

---

## 設立停損點

今天的 Android 修復終於完成，fix url PR pending in review，卸下心頭一件大事。

這次的經驗讓你意識到：**AI debug 是有停損點的。** 如果 AI 找 bug 的時間太久、或是產出不如預期，應該要衡量「改成自己查 document、自己執行」是否反而更快。AI 是工具，不是唯一解。

---

## 同事問你知不知道 NotebookLM

昨天同事問你知不知道 NotebookLM，在他問你之前你不知道這個工具。他的回應是：「你是想被淘汰？」

你一時無語。晚上你去研究了 NotebookLM 和 Claude Code 的整合方式，但目前還沒有太大的感覺。

你自己的判斷是：**AI 工具太多了，不知道某一個，不等於你要被淘汰。** 這樣嚇自己沒有必要。雖然心情有小小被影響，但你沒有讓自己被那句話帶著走。

---

## 你發現自己改變了

如果是幾年前的你，被同事這樣說，可能會難過、自我懷疑好久。

但現在的你，雖然還是小受影響，更多的是先回到自己：**先評估自己的能力與這件事的關係，而不是直接被別人的評價定義。**

你覺得這是一種成長。
